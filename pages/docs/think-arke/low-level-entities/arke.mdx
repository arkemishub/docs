## Persistance
When you create an Arke their information are stored inside our Database, according with the defined project, inside
the `arke_unit` table.

| Column name        | Description |
| :------------ | :---------: |
| id | identifier - can be a recognizable key or autogenerated (UUID) |
| arke_id | identifier of arke (`arke` if is the primitive) |
| data | the `parameters` list associated  |
| metadata | additional system information  |
| inserted_at | when arke item is created  |
| updated_at | when arke is updated  |

## RESTful Endpoints

When you create an Arke a set of default API are generated for your project, their could be authenticated and
additional params are ready-to-use like sorting, filtering and more

### Endpoints

| Name | Method       | Path | Description |
| :---: | :------------ | :---: | :---------: |
| Base Struct | GET | `{{base_path}}`/arke/struct | Get the list of parameters with their type (base structure) |
| Get All | GET | `{{base_path}}`/arke/unit | Get all arke |
| Get | GET | `{{base_path}}`/arke/unit/`{{arke_id}}` | Get specific arke ID |
| ... |

##### Headers

To authenticate your request use the authorization and the reference project

| Name | Value | Description |
| :---: | :------------ | :---: |
| Authorization | `{{token}}` | Authorization token to authenticate the api |
| arke-project-key | `{{project}}` | The identifier project |

##### Params

Use the query params to filter, sort and manage your response

| Name | Type | Description | Detail |
| :---: | :------------ | :---: |  :---: |
| offset | number | Start offset for pagination |
| limit | number | Count limit for pagination |
| order | string | Sorting definition ex. `asc;updated_at` | [Discover more](/arke/sorting)
| filter | string | Filter the response with condition ex. `eq(name,test)` | [Discover more](/arke/filters)
